# weeklyreport — 周报生成器

根据最终版第二十六次汇报的格式生成 `{yyMMdd}-栾歆瑶第N次汇报.docx` 周报文档。

## 使用方式

对话中说 **"写周报"** 或 **"/weeklyreport"**。系统会收集三部分信息：

1. **项目工作任务** — 参与了什么项目？做了什么修改？
2. **学习工作** — 学习了什么新内容？有什么理解和应用？
3. **文献阅读** — 读了什么论文？（ABS三星+/一区，关键词与成品油/配送/多目标优化相关）

## 文档格式

| 元素 | 格式 |
|------|------|
| 标题 | 黑体 18pt 加粗居中，前后段距 8pt |
| 章节标题（一、二、三…） | 宋体小四（12pt）加粗，首行缩进 24pt，1.5 倍行距 |
| 正文 | 宋体小四（12pt），首行缩进 24pt，1.5 倍行距 |
| 子项 `(1)` | 编号非加粗 + 标题加粗 + 内容普通 |
| 文献编号项 `(1)~(7)` | 编号非加粗 + 标签加粗 + 内容普通 |
| 英文论文名后自动加中文译名 | 纯文本无前缀 |
| 空白行 | 全文无空白行 |

## 文档结构

### 一、项目工作任务
- 简洁的正文段落，每项以 `①` 开头
- 附带个人理解与学习

### 二、学习工作
- `（1）[non-bold]下载[bold]与安装[bold]流程[normal]` 格式
- 详细描述 + 自己的理解

### 三、文献阅读
- 论文名称（纯文本）→ 中文译名（纯文本）→ 期刊 → 年份 → 作者
- `（1）关键词：[bold]` `（2）研究背景：[bold]` `（3）研究方法：[bold]` `（4）模型介绍：[bold]` `（5）实验设计：[bold]`
- `（6）研究结论：[bold]` 分条列出
- `（7）[non-bold]收获[bold]` 取代"对毕业论文的启发"

```python
        "year": "年份",

        "numbered_items": [

        ],
        "conclusion": ["结论1", "结论2", ...],
        "harvest": "收获内容（取代启发部分）",
    },

```
